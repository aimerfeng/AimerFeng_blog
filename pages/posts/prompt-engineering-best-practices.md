## 为什么提示词很重要

大语言模型的输出质量很大程度上取决于输入的提示词。同样的模型，不同的提示词可能产生天差地别的结果。

## 提示词的基本结构

一个好的提示词通常包含以下要素：

### 1. 角色设定（Role）

告诉 AI 它应该扮演什么角色：

```
你是一位资深的前端工程师，专注于 Vue.js 和 React 开发...
```

### 2. 任务描述（Task）

清晰地说明你想要 AI 做什么：

```
请帮我审查以下代码，找出潜在的性能问题和安全漏洞...
```

### 3. 上下文信息（Context）

提供必要的背景信息：

```
这是一个电商网站的购物车模块，使用 Vue 3 + Pinia 开发...
```

### 4. 输出格式（Format）

指定期望的输出格式：

```
请以 Markdown 格式输出，包含问题描述、严重程度和修复建议...
```

### 5. 约束条件（Constraints）

添加必要的限制：

```
代码示例使用 TypeScript，不要使用任何第三方库...
```

## 实用技巧

### 使用分隔符

用清晰的分隔符区分不同部分：

```
### 任务
[任务描述]

### 输入
[输入内容]

### 要求
[具体要求]
```

### 提供示例

Few-shot learning 效果显著：

```
示例输入: "hello world"
示例输出: "Hello World"

现在处理: "prompt engineering"
```

### 链式思考

让 AI 逐步推理：

```
请一步一步分析这个问题：
1. 首先，识别问题的核心
2. 然后，列出可能的解决方案
3. 最后，给出最佳建议
```

## 常见错误

1. **过于模糊**: "帮我写代码" → "帮我用 Python 写一个读取 CSV 文件的函数"
2. **缺少上下文**: 没有说明技术栈、版本、使用场景
3. **期望不明确**: 没有指定输出格式和长度

好的提示词 = 明确的角色 + 清晰的任务 + 充足的上下文 + 具体的格式要求

掌握这些技巧，你就能更高效地使用 AI 工具。
